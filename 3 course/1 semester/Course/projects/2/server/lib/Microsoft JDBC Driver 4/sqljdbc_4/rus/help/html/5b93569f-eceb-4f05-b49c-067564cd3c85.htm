Работа с большими объемами данных

Драйвер JDBC реализует поддержку адаптивной буферизации, которая позволяет получать любые данные большого объема, не расходуя ресурсы на серверные курсоры. При адаптивной буферизации драйвер Драйвер Microsoft JDBC для SQL Server получает результаты выполнения инструкции из экземпляра SQL Server по мере того, как их запрашивает приложение, а не все сразу. Драйвер также удаляет результаты, когда приложение теряет к ним доступ.

В версии 1.2 драйвера Microsoft SQL Server 2005 JDBC по умолчанию использовался режим буферизации full. Если приложение не устанавливало свойство responseBuffering в значение adaptive в свойствах соединения или с помощью метода setResponseBuffering объекта SQLServerStatement, то драйвер поддерживал одновременное считывание всех результатов с сервера. Чтобы включить режим адаптивной буферизации, приложение должно было явно устанавливать свойство соединения responseBuffering в значение adaptive.

Режим буферизации по умолчанию — adaptive, когда драйвер JDBC выполняет буферизацию для минимально необходимого объема данных. Дополнительные сведения об использовании адаптивной буферизации см. в разделе Использование адаптивной буферизации.
