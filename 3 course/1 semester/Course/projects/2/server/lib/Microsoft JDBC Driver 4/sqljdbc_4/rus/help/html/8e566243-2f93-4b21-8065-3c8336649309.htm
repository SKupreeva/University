Использование SSL-шифрования

SSL-шифрование поддерживает передачу зашифрованных данных по сети между экземпляром SQL Server и клиентским приложением.

Шифрование SSL ― это протокол для установления безопасного канала связи и предотвращения перехвата критических или конфиденциальных сведений в сети и по другим каналам связи. Шифрование SSL позволяет клиенту и серверу выполнить взаимную проверку подлинности. После проверки подлинности шифрование SSL обеспечивает зашифрованное соединение между участниками канала для безопасной передачи сообщений.

Драйвер Драйвер Microsoft JDBC для SQL Server обеспечивает инфраструктуру включения и выключения шифрования для определенного соединения, исходя из определенных пользователем свойств и параметров сервера и клиента. Пользователь может указать место хранения сертификата и пароль, имя узла, используемого для проверки сертификата, а также когда следует шифровать канал.

SSL-шифрование повышает безопасность передачи данных по сетям между экземплярами SQL Server и приложениями. Включение шифрования не снижает производительности.

В подразделах данного раздела описано, каким образом версия драйвера Драйвер Microsoft JDBC для SQL Server поддерживает SSL-шифрование, включая новые свойства соединения, а также настройку доверенного хранилища на стороне клиента.

Рекомендуется использовать свойство hostNameInCertificate для проверки SSL-сертификатов.
