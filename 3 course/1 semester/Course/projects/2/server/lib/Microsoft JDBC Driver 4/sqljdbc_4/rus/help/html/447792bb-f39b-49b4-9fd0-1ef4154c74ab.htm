Системные требования для драйверов JDBC

Чтобы получить доступ к данным из базы данных SQL Server с помощью драйвера Драйвер Microsoft JDBC для SQL Server, на компьютере должны быть установлены следующие компоненты:

  • Драйвер Microsoft JDBC для SQL Server

  • Среда выполнения Java

Требования к среде выполнения Java

Начиная с версии Драйвер Microsoft JDBC для SQL Server, в драйвере JDBC поддержка API спецификации Java Database Connectivity была расширена и теперь включает API JDBC 4.0. JDBC 4.0 была реализована как часть Sun Java SE Development Kit (JDK) 6.0 и среды выполнения Java (JRE) 6.0. JDBC 4.0 является множеством JDBC 3.0.

Драйвер Драйвер Microsoft JDBC для SQL Server поддерживает JDBC 3.0 и JDBC 4.0.

При развертывании драйвера Драйвер Microsoft JDBC для SQL Server в операционных системах Windows и UNIX необходимо использовать пакеты установки sqljdbc_<version>_enu.exe и sqljdbc_<version>_enu.tar.gz соответственно. Дополнительные сведения о развертывании драйвера JDBC см. в разделе Развертывание драйвера JDBC.

Драйвер JDBC во всех пакетах установки содержит две JAR-библиотеки классов, sqljdbc.jar и sqljdbc4.jar, что позволяет обеспечить поддержку обратной совместимости и возможных вариантов обновления.

JAR

Описание

sqljdbc.jar

Библиотека классов sqljdbc.jar обеспечивает поддержку JDBC 3.0.

Для класса sqljdbc.jar необходима среда выполнения Java (JRE) версии 5.0. Использование sqljdbc.jar в JRE 6.0 приведет к возникновению исключения при соединении с базой данных.

Драйвер JDBC не поддерживает JRE 1.4. При использовании драйвера JDBC необходимо обновить JRE 1.4 до версии 5.0 или 6.0. В некоторых случаях может потребоваться перекомпиляция приложения, если оно не совместимо с JDK 5.0 или более поздней версией. Дополнительные сведения см. в документации на веб-сайте Sun Microsystems.

sqljdbc4.jar

Библиотека классов sqljdbc4.jar обеспечивает поддержку JDBC 4.0. Она включает все функции sqljdbc.jar, а также новые методы JDBC 4.0.

Для класса sqljdbc4.jar необходима среда выполнения Java (JRE) версии 6.0. Использование sqljdbc4.jar в JRE 1.4 или 5.0 приведет к возникновению исключения.

Если необходимо использование приложения в JRE 6.0, следует использовать sqljdbc4.jar, даже если в приложении не используются функции JDBC 4.0.

Обратите внимание, что драйвер JDBC разработан и поддерживается при использовании со всеми основными аналогами виртуальных машин Java от Sun, но его проверка выполняется в Sun JRE 5.0 или более поздней версии.

Требования к SQL Server

Драйвер JDBC поддерживает подключения к SQL Server 2005 и более поздним версиям.

Требования к операционной системе

Драйвер JDBC предназначен для работы в любой операционной системе, поддерживающей использование виртуальной машины Java (JVM). Официально были протестированы только следующие операционные системы: Sun Solaris, SUSE Linux, Windows Vista и Windows 7.

Поддерживаемые языки

Драйвер JDBC поддерживает все параметры сортировки столбцов SQL Server. Дополнительные сведения о параметрах сортировки, поддерживаемых драйвером JDBC, см. в разделе Функции поддержки различных языков драйвера JDBC.

Дополнительные сведения о параметрах сортировки см. в разделе "Использование параметров сортировки" электронной документации SQL Server.
