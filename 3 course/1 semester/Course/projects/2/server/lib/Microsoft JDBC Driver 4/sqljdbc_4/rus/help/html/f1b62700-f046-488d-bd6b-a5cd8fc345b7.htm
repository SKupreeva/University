Задание свойств соединения

Свойства строки подключения могут быть заданы различными способами.

  • В виде пар «имя=значение» в URL-адресе соединения при подключении с помощью класса DriverManager.

  • В виде пар «имя=значение» в параметре Properties метода Connect класса DriverManager.

  • С помощью аргумента соответствующего метода задания свойств источника данных драйвера. Например:

    datasource.setServerName(value)
    datasource.setDatabaseName(value)

Примечания

В именах свойств не учитывается регистр символов. Дублирующиеся имена свойств разрешаются в следующем порядке:

  1. аргументы API (например, имя пользователя и пароль);

  2. Коллекция свойств.

  3. Последний экземпляр в строке подключения.

Кроме того, разрешено использование свойств с неизвестными драйверу именами. Их значения не проверяются драйвером JDBC на предмет учета регистра символов.

Допускается использование синонимов, которые разрешаются в порядке очередности, аналогично повторяющимся именам свойств.

В следующей таблице перечислены все доступные на данный момент свойства строки подключения для драйвера JDBC.

String

NULL

Имя базы данных для соединения. Если не установлено, подключение выполняется к базе данных по умолчанию.

disableStatementPooling

boolean

["true"|"false"]

true

Сейчас поддерживается только значение «true». Если задать значение «false», будет вызвано исключение.

encrypt

boolean

["true"|"false"]

false

Установите значение "true", чтобы указать, что SQL Server использует шифрование SSL для всех данных, посланных между клиентом и сервером, при условии, что у сервера есть установленный сертификат. Значение по умолчанию — «false».

failoverPartner

String

NULL

Имя сервера отработки отказа, используемого в конфигурации зеркального отображения базы данных. Это свойство используется при ошибке первоначального подключения к основному серверу. После выполнения первоначального подключения это свойство игнорируется. Должно использоваться совместно со свойством databaseName.

Этот драйвер не поддерживает возможность указания номера порта экземпляра сервера для экземляра партнера по обеспечению обработки отказа в виде части свойства failoverPartner в строке подключения. Однако поддерживается задание в одной строке подключения свойств serverName, instanceName, portNumber экземпляра основного сервера и свойства failoverPartner экземляра партнера по обеспечению обработки отказа.

Если в свойстве соединения Server указано имя виртуальной сети, то использовать зеркальное отображение базы данных нельзя. Дополнительные сведения см. в разделе Поддержка высокой доступности и аварийного восстановления в драйвере JDBC.

hostNameInCertificate

String

NULL

Имя узла, используемого для проверки SSL-сертификата SQL Server.

Если свойство hostNameInCertificate не задано или имеет значение NULL, драйвер JDBC Microsoft Драйвер Microsoft JDBC для SQL Server будет использовать значение свойства serverName в URL-адресе соединения в качестве имени узла для проверки SSL-сертификата SQL Server.

Это свойство используется в сочетании со свойством encrypt и свойством trustServerCertificate. Это свойство оказывает влияние на проверку сертификата только в том случае, если свойство encrypt имеет значение true, а свойство trustServerCertificate имеет значение false. Чтобы операция была выполнена успешно, убедитесь, что значение, переданное в hostNameInCertificate, в точности совпадает с общим именем (CN) или DNS-именем в SAN в сертификате сервера для соединения по протоколу SSL. Дополнительные сведения см. в разделе Основные сведения о поддержке SSL.

instanceName

String

[<=128 символов]

NULL

Имя экземпляра SQL Server для соединения. Если не задано, подключение выполняется к экземпляру по умолчанию. В случае, когда указан порт и instanceName, см. примечания к порту.

Если в свойстве соединения Server указано имя виртуальной сети, то использовать свойство соединения instanceName нельзя. Дополнительные сведения см. в разделе Поддержка высокой доступности и аварийного восстановления в драйвере JDBC.

integratedSecurity

boolean

["true"|"false"]

false

Установите значение true, чтобы служба SQL Server использовала учетные данные Windows для проверки подлинности пользователя или приложения. Если установлено значение true, драйвер JDBC выполняет поиск учетных данных, которые уже были предоставлены при входе в сеть или на компьютер, в локальном кэше учетных данных компьютера. Если установлено значение false, необходимо будет предоставить имя пользователя и пароль.

Данное свойство соединения поддерживается только в операционных системах Windows Microsoft.

lastUpdateCount

boolean

["true"|"false"]

true

Если установлено значение true, возвращается только последнее значение счетчика обновлений из инструкции SQL, переданной серверу. Оно может быть использовано в одной из инструкций SELECT, INSERT, DELETE, чтобы игнорировать последующие изменения счетчика обновлений, вызываемые триггерами сервера. Установка для этого свойства значения false влечет за собой возврат всех значений счетчика обновлений, включая изменения, вызванные триггерами сервера.

Это свойство применяется только при использовании с методами executeUpdate. Все другие методы execute возвращают все результаты и значения счетчика обновлений.

Это свойство влияет только на значения счетчика обновлений, возвращаемые триггерами сервера. Оно не влияет на результирующие наборы или ошибки, возникшие в процессе выполнения триггеров.

lockTimeout

int

-1

Время в миллисекундах, по истечении которого база данных сообщает об истечении времени ожидания блокировки. Режим по умолчанию — время ожидания неограничено. Если задано, это значение по умолчанию для всех инструкций в подключении. Обратите внимание, что Statement.setQueryTimeout() может быть использован для установки времени ожидания для определенных инструкций. Значение может быть равным 0 — время ожидания отсутствует.

loginTimeout

int [0..65535]

15

Время в секундах, по истечении которого драйвер инициирует превышение времени ожидания неудачного подключения. Нулевое значение указывает, что время ожидания равно системному времени ожидания по умолчанию, которое по умолчанию составляет 15 секунд. Ненулевое значение представляет время в секундах, по истечении которого драйвер инициирует превышение времени ожидания неудачного подключения.

Если в свойстве соединения Server указано имя виртуальной сети, то необходимо задать значение ожидания три минуты или более, чтобы была возможна отработка отказа при соединении. Дополнительные сведения см. в разделе Поддержка высокой доступности и аварийного восстановления в драйвере JDBC.

multiSubnetFailover

Boolean

false

Всегда указывайте multiSubnetFailover=true при соединении с прослушивателем группы доступности SQL Server 2012 или экземпляра отказоустойчивого кластера SQL Server 2012. Значение свойства multiSubnetFailover=true настраивает Драйвер Microsoft JDBC для SQL Server на ускоренное обнаружение и подключение к серверу, активному в данный момент. Допустимые значения — true и false. Дополнительные сведения см. в разделе Поддержка высокой доступности и аварийного восстановления в драйвере JDBC.

Доступ к свойству соединения multiSubnetFailover программным способом возможен через свойства getPropertyInfo, getMultiSubnetFailover и setMultiSubnetFailover.

packetSize

int [-1| 0 | 512..32767]

8000

Размер (в байтах) сетевого пакета, используемого для обмена данными с SQL Server. Значение -1 указывает на использование размера сетевого пакета сервера по умолчанию. Значение 0 указывает на использование максимального значения — 32767. Если этому свойству задано значение за пределами допустимого диапазона значений, возникнет исключение.

Не рекомендуется использовать свойство packetSize при включенном шифровании (encrypt=true). В противном случае в работе драйвера может быть вызвана ошибка подключения. Дополнительные сведения см. в описании метода setPacketSize класса SQLServerDataSource.

Пароль базы данных.

portNumber, порт

int [0..65535]

1433

Порт, который прослушивает SQL Server. Если номер порта задан в строке подключения, запросы к sqlbrowser не выполняются. В случае, когда указан порт и instanceName, подключение выполняется к указанному порту. Однако при этом выполняется проверка instanceName, и, если значение этого параметра не соответствует порту, возникает ошибка.

Рекомендуется всегда указывать номер порта, так как это более безопасно, чем использование sqlbrowser.

responseBuffering

String

["full"|"adaptive"]@@@

adaptive

Если свойство имеет значение adaptive, это указывает на буферизацию необходимого минимума данных. По умолчанию используется режим adaptive.

Если свойство имеет значение full, при выполнении инструкции выполняется чтение с сервера результирующего набора полностью.

После обновления драйвера JDBC с версии 1.2 по умолчанию будет использоваться режим буферизации adaptive. Если приложение никогда не задает свойство responseBuffering, и необходимо сохранить в приложении режим по умолчанию как в версии 1.2, необходимо задать свойству responseBufferring значение full в свойствах соединения или с помощью метода setResponseBuffering объекта SQLServerStatement.

selectMethod

String

["direct"|"cursor"]@@@

direct

Если этому свойству задано значение cursor, то для каждого созданного в этом соединении запроса создается курсор базы данных с параметрами TYPE_FORWARD_ONLY, CONCUR_READ_ONLY. Это свойство обычно необходимо, только если приложение создает очень большие результирующие наборы, которые не могут полностью находится в клиентской памяти. Если свойство имеет значение cursor, в клиентской памяти сохраняется ограниченное число строк результирующего набора. Режим по умолчанию — все строки результирующего набора сохраняются в клиентской памяти. Такой вариант обеспечивает наибольшую производительность, когда приложение обрабатывает все строки.

sendStringParametersAsUnicode

boolean

["true"|"false"]

true

Если свойство sendStringParametersAsUnicode имеет значение true, то строковые параметры отправляются на сервер в формате Юникода.

Если свойство sendStringParametersAsUnicode имеет значение false, то строковые параметры отправляются на сервер в формате, отличном от Юникода, например в ASCII/MBCS.

Значение свойства sendStringParametersAsUnicode по умолчанию — true.

Свойство sendStringParametersAsUnicode проверяется только при отправке значения параметра, имеющего тип JDBC CHAR, VARCHAR или LONGVARCHAR. Новые методы национальных символов в JDBC 4.0, например, методы setNString, setNCharacterStream, setNClob классов SQLServerPreparedStatement и SQLServerCallableStatement, всегда выполняют отправку на сервер своих значений параметров в формате Юникод, независимо от значения свойства.

Чтобы добиться максимальной производительности при обработке типов данных JDBC CHAR, VARCHAR и LONGVARCHAR, приложение должно установить свойство sendStringParametersAsUnicode в значение false и пользоваться методами для работы без национальных символов setString, setCharacterStream и setClob классов SQLServerPreparedStatement и SQLServerCallableStatement.

Когда приложение задает свойству sendStringParametersAsUnicode значение false и использует метод, не поддерживающий национальные символы, для доступа на стороне сервера к типам данных в формате Юникод (например, nchar, nvarcharntext), часть данных может быть потеряна, если сортировка базы данных не поддерживает символы в строковых параметрах, которые передаются методом, не поддерживающим национальные символы.

Обратите внимание, что приложение должно использовать методы национальных символов setNString, setNCharacterStream, setNClob классов SQLServerPreparedStatement и SQLServerCallableStatement для следующих типов данных JDBC NCHAR, NVARCHAR, LONGNVARCHAR.

sendTimeAsDatetime

boolean

["true"|"false"]

true

Это свойство добавлено в версии 3.0 драйвера SQL Server JDBC.

Если задано значение true, то значения java.sql.Time будут отправляться на сервер как значения типа SQL Server datetime.

Если задано значение false, то значения java.sql.Time будут отправляться на сервер как значения типа SQL Server time.

Свойство sendTimeAsDatetime можно изменять программным образом с помощью SQLServerDataSource.setSendTimeAsDatetime.

Значение по умолчанию этого свойства в будущей версии может быть изменено.

Дополнительные сведения о том, как Драйвер Microsoft JDBC для SQL Server настраивает значения java.sql.Time перед их отправкой на сервер, см. в разделе Настройка способа отправки значений java.sql.Time на сервер.

NULL

Компьютер, на котором работает SQL Server.

Можно также указать имя виртуальной сети в группе доступности Группы доступности AlwaysOn. Дополнительные сведения см. в разделе Поддержка высокой доступности и аварийного восстановления в драйвере JDBC.

trustServerCertificate

boolean

["true"|"false"]

false

Установите значение в true, чтобы Драйвер Microsoft JDBC для SQL Server не проверял SSL-сертификат SQL Server.

Если значение равно true, SSL-сертификат SQL Server автоматически считается доверенным, когда для шифрования уровня связи используется SSL.

Если значение равно false, то Драйвер Microsoft JDBC для SQL Server будет выполнять проверку SSL-сертификата сервера. Если проверка сертификата сервера завершается ошибкой, в работе драйвера будет вызвана ошибка, и соединение будет разорвано. Значение по умолчанию — «false». Чтобы операция была выполнена успешно, убедитесь в том, что значение, переданное в serverName, в точности совпадает с общим именем (CN) или DNS-именем в SAN в сертификате сервера для соединения по протоколу SSL. Дополнительные сведения см. в разделе Основные сведения о поддержке SSL.

Это свойство используется в сочетании со свойством encrypt. Это свойство оказывает влияние на проверку SSL-сертификата сервера только в том случае, если свойство encrypt имеет значение true.

trustStore

String

NULL

Путь к файлу сертификата trustStore (включая имя файла). Файл trustStore содержит список сертификатов, которым доверяет клиент.

Когда свойство не задано или имеет значение null, драйвер будет использовать правила поиска фабрики диспетчеров доверия, чтобы определить, какой из хранящихся сертификатов необходимо использовать.

Используемая по умолчанию фабрика SunX509 TrustManagerFactory пытается найти доверенный ресурс, производя поиск в следующем порядке.

  • Файл, на который указывает системное свойство виртуальной машины Java (JVM) «javax.net.ssl.trustStore».

  • Файл «<домашний-каталог-java>/lib/security/jssecacerts».

  • Файл «<домашний-каталог-java>/lib/security/cacerts».

Дополнительные сведения см. в документации по интерфейсу SUNX509 TrustManager на веб-сайте компании Sun Microsystems.

Это свойство оказывает влияние на поиск сертификата trustStore только в том случае, если свойство encrypt имеет значение true, а свойство trustServerCertificate имеет значение false.

trustStorePassword

String

NULL

Пароль, используемый для проверки целостности данных trustStore.

Если задано свойство trustStore, а свойство trustStorePassword не задано, проверка целостности trustStore не выполняется.

Когда не заданы оба свойства trustStore и trustStorePassword, драйвер будет использовать системные свойства JVM «javax.net.ssl.trustStore» и «javax.net.ssl.trustStorePassword». Если не задано значение системного свойства "javax.net.ssl.trustStorePassword", проверка целостности trustStore не выполняется.

Если свойство trustStore не задано, а свойство trustStorePassword задано, драйвер JDBC будет использовать файл, заданный "javax.net.ssl.trustStore" в качестве доверенного хранилища, целостность доверенного хранилища проверяется с использованием указанного trustStorePassword. Это может потребоватьс в случае, если клиентское приложение не желает сохранять пароль с системеном свойстве JVM.

Свойство trustStorePassword оказывает влияние на поиск сертификата trustStore только в том случае, если свойство encrypt имеет значение true и свойство trustServerCertificate имеет значение false.
