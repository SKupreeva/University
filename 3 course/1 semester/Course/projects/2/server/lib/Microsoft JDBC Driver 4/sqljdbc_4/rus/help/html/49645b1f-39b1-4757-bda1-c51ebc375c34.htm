Соединение с базой данных в SQL Azure

В настоящем разделе обсуждаются возможные проблемы при использовании Драйвер Microsoft JDBC для SQL Server для установления соединения с базой данных SQL Azure. Дополнительные сведения о базах данных SQL Azure см. в следующих разделах:

  • База данных SQL Azure

  • Как подключиться к SQL Azure с помощью JDBC

  • Использование SQL Azure в Java

Подробные сведения

В SQL Azure для вызова метода SQLServerDatabaseMetaData.getCatalogs необходимо установить соединение с базой данных master.

SQL Azure не поддерживает возврат всего набора каталогов из пользовательских баз данных. Метод SQLServerDatabaseMetaData.getCatalogs использует для получения каталогов представление sys.databases. Обсуждение разрешений см. в разделе sys.databases (база данных SQL Azure), содержащее описание работы SQLServerDatabaseMetaData.getCatalogs в SQL Azure.

Прерванные соединения

При установлении соединения с SQL Azure простаивающие соединения через определенное время бездействия могут быть разорваны сетевым компонентом (например, брандмауэром). Чтобы избежать разрыва связи простаивающих соединений сетевыми компонентами, необходимо задать следующие параметры в реестре операционной системе, в которой загружается драйвер (или их аналоги для операционных систем, отличных от Windows):

Параметр реестра

Рекомендуемое значение

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\KeepAliveTime

30000

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\KeepAliveInterval

1000

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\TcpMaxDataRetransmission

10

После этого необходимо перезагрузить компьютер, чтобы новые параметры вступили в силу.

Чтобы эта операция выполнялась при запуске Windows Azure, создайте выполняемую при запуске задачу, которая будет добавлять значения в реестр. Например, в файл определения службы добавьте следующую задачу, выполняемую при запуске.

<Startup>
    <Task commandLine="AddKeepAlive.cmd" executionContext="elevated" taskType="simple">
    </Task>
</Startup>

Затем добавьте файл AddKeepAlive.cmd в проект. Выберите для параметра «Копировать в выходной каталог» значение «Всегда копировать». Ниже приведен образец файла AddKeepAlive.cmd.

if exist keepalive.txt goto done
time /t > keepalive.txt
REM Workaround for JDBC keep alive on SQL Azure
REG ADD HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters /v KeepAliveTime /t REG_DWORD /d 30000 >> keepalive.txt
REG ADD HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters /v KeepAliveInterval /t REG_DWORD /d 1000 >> keepalive.txt
REG ADD HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters /v TcpMaxDataRetransmission /t REG_DWORD /d 10 >> keepalive.txt
shutdown /r /t 1
:done
Добавление имени сервера к идентификатору пользователя UserId в строке подключения

До версии 4.0 Драйвер Microsoft JDBC для SQL Server при установлении соединения с базой данных SQL Azure требовалось добавление имени сервера к параметру UserId в строке подключения (например, «пользователь@имя_сервера»). Начиная с версии 4.0 Драйвер Microsoft JDBC для SQL Server добавление компонента «@имя_сервера» к параметру UserId в строке подключения не требуется.

Для использования шифрования необходимо задать hostNameInCertificate

При установлении соединения с SQL Azure необходимо задать hostNameInCertificate, если указано encrypt=true. (если имя сервера в строке подключения имеет вид shortName.domainName, то установите свойство hostNameInCertificate в значение *.domainName).

Например:

jdbc:sqlserver://abcd.int.mscds.com;databaseName= myDatabase;user=myName;password=myPassword;encrypt=true;hostNameInCertificate= *.int.mscds.com;
