Новые возможности драйвера JDBC

Драйвер Драйвер Microsoft JDBC 4.0 для SQL Server содержит несколько новых и улучшенных функций.

Сведения о подключении к базе данных SQL Azure

Теперь имеется раздел со сведениями о подключении к базе данных SQL Azure. Дополнительные сведения см. в разделе Соединение с базой данных в SQL Azure.

Поддержка аварийного восстановления с высоким уровнем доступности (HADR)

Драйвер Драйвер Microsoft JDBC 4.0 для SQL Server поддерживает аварийное восстановление с высоким уровнем доступности благодаря возможности соединения с группами доступности AlwaysOn в SQL Server 2012. Дополнительные сведения см. в разделе Поддержка высокой доступности и аварийного восстановления в драйвере JDBC.

Использование встроенной проверки подлинности по протоколу Kerberos для соединения с SQL Server

Драйвер Драйвер Microsoft JDBC 4.0 для SQL Server реализует встроенную поддержку подлинности Kerberos типа 4 для приложений, устанавливающих соединение с базами данных SQL Server. Дополнительные сведения см. в разделе Использование встроенной проверки подлинности по протоколу Kerberos для соединения с SQL Server. (Встроенная проверка подлинности Kerberos типа 2 доступна в версиях Драйвер Microsoft JDBC для SQL Server ранее 4.0.)

Доступ к диагностической информации в расширенном журнале событий

Сведения в расширенном журнале событий сервера могут оказаться полезными при выявлении причины ошибок соединения. Дополнительные сведения см. в разделе Доступ к диагностической информации в журнале расширенных событий.

Дополнительная поддержка разреженных столбцов

Если приложение уже использует данные из таблицы, в которой есть разреженные столбцы, то можно ожидать увеличения производительности. Сведения о столбцах (включая сведения о разреженных столбцах) можно получить с помощью Метод getColumns (SQLServerDatabaseMetaData). Дополнительные сведения о SQL Server разреженных столбцах см. в разделе Использование разреженных столбцов.

Метод isSparseColumnSet добавлен в класс SQLServerResultSetMetaData, чтобы указывать, является ли столбец из результирующего набора набором разреженных столбцов.

Xid.getFormatId

До версии Драйвер Microsoft JDBC 4.0 для SQL Server драйвер JDBC получал идентификатор формата от приложения, но не передавал это значение серверу баз данных. Вместо этого драйвер JDBC передавал серверу баз данных 0 в качестве значения идентификатора формата.

Начиная с версии Драйвер Microsoft JDBC 4.0 для SQL Server, драйвер JDBC передает идентификатор формата от приложения серверу баз данных. Чтобы обеспечить такое поведение драйвера, необходимо обновить файл sqljdbc_xa.dll на сервере. Дополнительные сведения о копировании обновленной версии файла sqljdbc_xa.dll на сервер см. в разделе Основные сведения о транзакциях XA.

Процессор Itanium не поддерживается для приложений драйвера JDBC 4.0

Работа приложений Драйвер Microsoft JDBC 4.0 для SQL Server на компьютерах Itanium не поддерживается.
