Элементы SQLServerXADataSource

В следующих таблицах перечислены элементы, предоставляемые классом SQLServerXADataSource.

Конструкторы

Имя

Описание

SQLServerXADataSource ()

Инициализирует новый экземпляр класса SQLServerXADataSource.

Поля

None.

Наследуемые поля

Нет.

Методы

Имя

Описание

getApplicationIntent

(Наследуется от SQLServerDataSource) Возвращает значение свойства соединения applicationIntent.

getApplicationName

(Наследуется от SQLServerDataSource) Возвращает имя приложения.

getConnection

(Наследуется от SQLServerDataSource) Устанавливает соединение с источником данных, который представляется этим объектом DataSource.

getDatabaseName

(Наследуется от SQLServerDataSource) Возвращает имя базы данных.

getFailoverPartner

(Наследуется от SQLServerDataSource) Возвращает имя сервера отработки отказа, которое используется в настройках зеркального отображения баз данных.

getInstanceName

(Наследуется от SQLServerDataSource) Возвращает имя экземпляра SQL Server.

getLastUpdateCount

(Наследуется от SQLServerDataSource) Возвращает значение типа Boolean, которое указывает, включено ли свойство lastUpdateCount.

getLockTimeout

(Наследуется от SQLServerDataSource) Возвращает значение типа int, которое указывает продолжительность ожидания (в мс) до того, как база данных сообщит об истечении времени ожидания блокировки.

getLoginTimeout

(Наследуется от SQLServerDataSource) Возвращает продолжительность ожидания (в с) объекта DataSource при попытке установить соединение.

getLogWriter

(Наследуется от SQLServerDataSource) Возвращает символьный выходной поток, который используется для всех сообщений ведения журнала и трассировки.

getMultiSubnetFailover

(Наследуется от класса SQLServerDataSource) Возвращает значение свойства соединения multiSubnetFailover.

getPooledConnection

(Наследуется от SQLServerConnectionPoolDataSource) Устанавливает физическое подключение к базе данных, которое может использоваться как соединение в пуле.

getPortNumber

(Наследуется от SQLServerDataSource) Возвращает номер текущего порта, который используется для взаимодействия с SQL Server.

getReference

Возвращает ссылку на этот объект SQLServerXADataSource.

getSelectMethod

(Наследуется от SQLServerDataSource) Возвращает тип курсора по умолчанию, который используется для всех результирующих наборов, создаваемых с помощью этого объекта DataSource.

getSendStringParametersAsUnicode

(Наследуется от SQLServerDataSource) Возвращает значение типа Boolean, которое указывает, включена ли отправка параметров типа String на сервер в формате Юникод.

getServerName

(Наследуется от SQLServerDataSource) Возвращает имя компьютера, на котором выполняется SQL Server.

getURL

(Наследуется от SQLServerDataSource) Возвращает URL-адрес, который используется при подключении к источнику данных.

getUser

getWorkstationID

(Наследуется от SQLServerDataSource) Возвращает имя клиентского компьютера, которое используется при подключении к источнику данных.

getXAConnection

Устанавливает физическое подключение к базе данных, которое будет использоваться в распределенной транзакции.

getXopenStates

(Наследуется от SQLServerDataSource) Возвращает значение типа Boolean, которое указывает, включено ли преобразование состояний SQL в состояния, соответствующие стандарту XOPEN.
