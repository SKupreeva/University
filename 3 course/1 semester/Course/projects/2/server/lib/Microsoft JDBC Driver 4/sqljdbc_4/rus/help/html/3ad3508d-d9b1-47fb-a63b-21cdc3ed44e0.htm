Развертывание драйвера JDBC

При развертывании приложения, зависящего от драйвера Драйвер Microsoft JDBC для SQL Server, необходимо вместе с приложением распространять драйвер JDBC. В отличие от компонентов доступа к данным Windows DAC, которые являются компонентами операционной системы Windows, драйвер JDBC считается компонентом SQL Server.

Хотя можно свободно выполнить распределение драйвера JDBC в соответствии с требованиями приложения, необходимо сначала зарегистрироваться. Зарегистрироваться для распространения драйвера JDBC можно на странице Регистрация для получения прав на распространение драйвера Microsoft JDBC для SQL Server.

Для развертывания драйвера JDBC в приложении используются два подхода. При использовании одного файлы драйвера JDBC включаются как часть собственного настраиваемого пакета установки. При втором подходе используется предоставляемый Майкрософт пакет установки JDBC, который можно загрузить в Центре разработки драйвера Microsoft JDBC для SQL Server.

В следующих разделах описывается использование пакета установки в операционных системах Windows и UNIX.

Дополнительные общие сведения о развертывании приложений Java см. на веб-сайте Java.

Развертывание драйвера JDBC в системах Windows

При развертывании драйвера JDBC в операционных системах Windows необходимо использовать исполняемый вариант ZIP-файла пакета установки, который обычно называется sqljdbc_<version>_<language>.exe.

Для автоматического запуска исполняемого файла в формате zip необходимо использовать параметр командной строки /auto в командной строке или в пакетном файле, как описано ниже:

sqljdbc_<version>_<language>.exe /auto

При использовании параметра /auto установка не будет полностью автоматической, поскольку на экране пользователя все же будет отображаться диалоговое окно WinZip. Однако оно не требует выполнения каких-либо действий пользователя и закрывается после выполнения операции распаковки.

Развертывание драйвера в системах UNIX

При развертывании драйвера драйвера JDBC в операционных системах UNIX необходимо использовать исполняемый файл в формате gzip пакета установки, который обычно называется sqljdbc_<version>_<language>.tar.gz.

До установки драйвера JDBC необходимо убедиться в том, что в системе пользователя установлены программы gzip и tar, и что папки, содержащие исполняемые файлы обеих программ, добавлены к переменной среды PATH.

Чтобы распаковать сжатый TAR-файл, перейдите в каталог, в который нужно распаковать драйвер, и введите следующую команду:

gzip -d sqljdbc_<version>_<language>.tar.gz

Чтобы распаковать TAR-файл, переместите его в каталог, в который нужно установить драйвер, и введите следующую команду:

tar –xf sqljdbc_<version>_<language>.tar
