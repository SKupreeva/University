Использование метаданных о параметрах

Для выполнения запроса к объекту SQLServerPreparedStatement или SQLServerCallableStatement относительно параметров, которые он содержит, драйвер Драйвер Microsoft JDBC для SQL Server реализует класс SQLServerParameterMetaData. Этот класс содержит многочисленные поля и методы, которые возвращают сведения в виде одного значения.

Для создания объекта SQLServerParameterMetaData можно использовать методы getParameterMetaData классов SQLServerPreparedStatement и SQLServerCallableStatement.

В следующем примере открытое соединение с образцом базы данных SQL Server 2005 AdventureWorks передается в функцию, метод getParameterMetaData класса SQLServerCallableStatement используется для возвращения объекта SQLServerParameterMetaData, далее различные методы объекта SQLServerParameterMetaData используются для отображения сведений о типе и режиме параметров, которые содержатся в хранимой процедуре HumanResources.uspUpdateEmployeeHireInfo.

public static void getParameterMetaData(Connection con) {
   try {
      CallableStatement cstmt = con.prepareCall("{call HumanResources.uspUpdateEmployeeHireInfo(?, ?, ?, ?, ?)}");
      ParameterMetaData pmd = cstmt.getParameterMetaData();
      int count = pmd.getParameterCount();
      for (int i = 1; i <= count; i++) {
         System.out.println("TYPE: " + pmd.getParameterTypeName(i) + " MODE: " + pmd.getParameterMode(i));
      }
      cstmt.close();
   }
   catch (Exception e) {
      e.printStackTrace();
   }
}

Существуют определенные ограничения при использовании класса SQLServerParameterMetaData с подготовленными инструкциями. Драйвер JDBC поддерживает инструкции SELECT, DELETE, INSERT и UPDATE. Однако эти инструкции не должны содержать вложенные запросы. Кроме того, драйвер JDBC также поддерживает использование предикатов IN, IS, LIKE. Для условий поиска драйвер JDBC поддерживает использование операторов сравнения IS, <, >, LIKE, >=, <=, NOT IN, <>, !>, !<, !=. Оператор FREETEXT не поддерживается.
