1.1           Ленточная диаграмма

Основным типом графического представления информации является ленточная диаграмма. Она отображает данные в вдоль линии с каждой точки данных, представленной в виде прямоугольника. Здесь мы приводим примеры использования указанных в верхней части этой страницы  фрейма данных и один столбец данных w1$vals.

Чтобы создать ленточную диаграмму этих данных используйте команду stripchart:

> help(stripchart)
> stripchart(w1$vals)

 

 

 

Рисунок 1.1 – Представление графика функцией stripchart()

 

Как вы можете видеть, этот график, примерно как голые кости. Нет названия, ни названия осей. Это только показывает, как данные выглядят, если бы их все поставить вдоль одной линии и выделить поле для каждой точки. Если вы хотите увидеть какие точки повторяются, введите в параметре method=»stack»  в функции stripchart().Method по умолчанию «overplot» вызывает такие моменты overplotted, но также возможно указать «jitter» колебание точки, или « stack « –  совпадающие точки в штабелях. Последний метод имеет смысл только для очень детализированных данных. 

> stripchart(w1$vals,method="stack")
> stripchart(w1$vals,method="jitter")

 

 

Рисунок 1.2 – Использование параметра «jitter» и «stack»

 

Для расположения графика вертикально необходимо в параметре функции method указать TRUE (по умолчанию False).

> stripchart(w1$vals,vertical=TRUE)
> stripchart(w1$vals,vertical=TRUE,method="jitter")

 

Рисунок 1.3 – Вертикальное расположение графика

Последний тип графика, который мы рассмотрим – это график нормальной вероятности. Этот график используется для определения, если ваши данные близки к нормальному распределению. Вы не можете быть уверены, что данные распределены нормально, но можно исключить, если он не является нормально распределенной. 2.1.1. Несколько наборов данных на одном участке

Основная задача данного раздела – построить несколько наборов данных на том же графике. Во многих случаях способ сделать это, создать начальный график и потом добавить дополнительную информацию (другие графики).

Например, чтобы построить двумерные данные функцией plot(), используемая для инициализации и создания графика, можно использовать функцию points(), которая используется для добавления дополнительных наборов данных в графике.

Сначала определите набор нормально распределенных случайных чисел, а затем построить их. (Это тот же набор данных используется в примерах, приведенных ниже.)

plot(x,y,xlab="Independent",ylab="Dependent",main="Random Stuff")
> x1 <- runif(8,15,25)
> y1 <- 2.5*x1 - 1.0 + runif(8,-6,6)
> points(x1,y1,col=2)

 

Рисунок 2.1 – График с дополнительным набором данных

 

Следует отметить, что и в предыдущем примере, цвет для второго набора точек данных устанавливается с помощью параметра сol. Вы можете попробовать различные номера, чтобы увидеть, какие цвета доступны. Для большинства установок, есть по крайней мере восемь вариантов от 1 до 8. Также обратите внимание, что в приведенном выше точки, например, нанесены в виде кругов. Символ, который используется можно изменить с помощью параметра  pch.

> x2 <- runif(8,15,25)
> y2 <- 2.5*x2 - 1.0 + runif(8,-6,6)
> points(x2,y2,col=3,pch=2)

 

Рисунок 2.2 – График с дополнительным набором данных

 

Еще один полезный совет, добавить легенду. Это может быть сделано с помощью команды legend(). Параметры функции, являются х и у координаты на графике, чтобы разместить легенду следует знать список меток в вашем графике. 

> plot(x,y,xlab="Independent",ylab="Dependent",main="Random Stuff")

> plot(x,y,xlab="Independent",ylab="Dependent",main="Random Stuff",xlim=c(0,30),ylim=c(0,100))

2.1.2. Ошибка Бары 

Еще одна распространенная задача, это добавить планки погрешностей в набор точек данных. Это может быть достигнуто с помощью команды arrows(). Команда arrows()   указываются две пары координат, то есть две пары х и у значений. Затем команда рисует линию между каждой парой и добавляет “arrow head” при заданной длине и углу наклона.

 

> plot(x,y,xlab="Independent",ylab="Dependent",main="Random Stuff")
> xHigh <- x
> yHigh <- y + abs(rnorm(10,sd=3.5))
> xLow <- x
> yLow <- y - abs(rnorm(10,sd=3.1))
> arrows(xHigh,yHigh,xLow,yLow,col=2,angle=90,length=0.1,code=3)

 

Рисунок 2.4 – График с ошибкой бары

Обратите внимание, что код опция используется для указания, где бары тянутся. Его значение может быть 1, 2 или 3. Если код 1 бары рисуются в пар, приведенных в первом аргументе. Если код 2, бары рисуются на одном конце линии. Если код 3 бары рисуются по обе стороны.

 

2.1.3. Добавление шума (jitter)

 

В предыдущем примере произведем искусственное смещение. Это обычная вещь для разработки графиков. Простой способ добиться этого – использовать jitter команды.

> numberWhite <- rhyper(400,4,5,3)
> numberChipped <- rhyper(400,2,7,3)
> par(mfrow=c(1,2))
> plot(numberWhite,numberChipped,xlab="Number White Marbles Drawn",
       ylab="Number Chipped Marbles Drawn",main="Pulling Marbles")
> plot(jitter(numberWhite),jitter(numberChipped),xlab="Number White Marbles Drawn",
       ylab="Number Chipped Marbles Drawn",main="Pulling Marbles With Jitter")

 

Рисунок 2.5 – График до и после добавлением шума

2.1.4. Несколько графиков на одном изображении

 

Следует отметить, что новая команда была использована в предыдущем примере. Команда par() может быть использован для установки различных параметров. В приведенном выше примере в mfrow был установлен. Участки расположены в массиве, где количество по умолчанию строк и столбцов равно одному. Параметр mfrow является вектор с двумя входами. Первая запись число строк изображений. Вторая запись –  это количество столбцов. 

> par(mfrow=c(2,3))
> boxplot(numberWhite,main="first plot")
> boxplot(numberChipped,main="second plot")
> plot(jitter(numberWhite),jitter(numberChipped),xlab="Number White Marbles Drawn",ylab="Number Chipped Marbles Drawn",main="Pulling Marbles With Jitter")
> hist(numberWhite,main="fourth plot")
> hist(numberChipped,main="fifth plot")
> mosaicplot(table(numberWhite,numberChipped),main="sixth plot")

2.1.6. Парные отношения

 

Количество отношения могут быть построены в одно время с помощью команды pairs(). Идея заключается в том, что вы даете ему матрицу или кадр данных, и команда создает график рассеяния всех комбинаций данных.

> uData <- rnorm(20)
> vData <- rnorm(20,mean=5)
> wData <- uData + 2*vData + rnorm(20,sd=0.5)
> xData <- -2*uData+rnorm(20,sd=0.1)
> yData <- 3*vData+rnorm(20,sd=2.5)
> d <- data.frame(u=uData,v=vData,w=wData,x=xData,y=yData)
> pairs(d)

2.2.1. Barplot 

 

Команда plot() будет пытаться произвести соответствующие участки, основываясь на типе данных. Данных, которые определены выше, числовые данные. Необходимо преобразовать данные в факторы, чтобы убедиться, что команды plot(), обрабатывает его соответствующим образом. Команда as.factor используется, чтобы преобразовать данные, как факторы и гарантирует, что R рассматривает его как дискретные данные.

> numberWhite <- rhyper(30,4,5,3)
> numberWhite <- as.factor(numberWhite)
> plot(numberWhite)

В этом случае R будет строить график как barplot. Команда barplot() также может быть использована для создания barplot графтка. Команда barplot() требуется вектор высоты, хотя, и вы не можете просто дать ему исходные данные. Частоты для команды barplot() могут быть легко вычислены с помощью команды table().

> numberWhite <- rhyper(30,4,5,3)
> totals <- table(numberWhite)
> totals
numberWhite
0  1  2  3
4 13 11  2
> barplot(totals,main="Number Draws",ylab="Frequency",xlab="Draws")

В предыдущем примере команда barplot используется для указания названия графика и метки для осей. Вы можете изменять метки, установив имена строк таблицы.

> totals <- table(numberWhite)
> rownames(totals) <- c("none","one","two","three")
> totals
numberWhite
none   one   two three
4      13    11     2
> barplot(totals,main="Number Draws",ylab="Frequency",xlab="Draws")

2.2.2. Мозаичные графики 

Мозаичные графики используются для отображения пропорций таблиц, которые делятся на два или более условных распределений. Предполагается, что вы знакомы с использованием таблиц в R (смотрите раздел двухсторонних таблиц для получения дополнительной информации).

Здесь мы будем использовать составленные наборы данных, для легкого понимания, что R делает. Фиктивный набор данных определяется ниже. Идея заключается в том, что шестнадцать детей в возрасте восьми лет проходят собеседование. Им задали два вопроса. Первый вопрос, "Вы верите в Деда Мороза." Если они отвечают утвердительно, то записывается термин "вера", в противном случае будет записано "Нет веры". Второй вопрос, есть ли у них старший брат, старшая сестра или нет старших братьев и сестер. Ответы, записываются как "старший брат", "старшая сестра", или "нет старших».

2.3.2. Несколько окон 

Команды Dev позволяют создавать и управлять несколькими графическими окнами. Вы можете создавать новые окна с помощью команды dev.new(), также вы можете выбрать, какое из этих окон сделать активным с помощью команды dev.set (). Команды dev.list (), dev.next (), и dev.prev () могут быть использованы для вывода списка графических устройств, которые доступны в данный момент.

В следующем примере создаются три устройства. Они перечислены, и различные графики создаются на разных устройствах.

2.3.3. Печать в файл 

Есть несколько способов, чтобы напечатать график в файл. Важно, иметь возможность работать с графическими окнами. Первый способ: использование команды dev.print. Эта команда будет печатать копию текущего активного устройства, а формат определяется аргументом device.

  > x = rnorm(100)
> hist(x)
> dev.print(device=png,width=200,"hist.png")
> 

Чтобы узнать, какие устройства доступны на вашей системе, используйте команду help().

> help(device)

Второй способ печати в файл является создание устройства таким же образом, как и создание графического устройства в предыдущем разделе. После того, как устройство будет создано и различные графические команды даны, устройство отключается, чтобы записать результаты в файл.

> png(file="hist.png")
> hist(x)
> rug(x,side=1)
> dev.off()

 2.3.4. Аннотация и форматирование 

У большинства графических функций есть возможность отключить генерацию осей с использованием axes = False . Оси могут быть затем добавлены с помощью функции axis, которая предоставляет огромное количество параметров.

В примере ниже представлено как двумерный набор случайных чисел генерируются и отражаются графически в виде диаграммы разброса. Оси добавлены, но горизонтальная ось расположена в центре данных, а не в нижней части рисунка. Обратите внимание, что горизонтальные и вертикальные оси будут добавлены отдельно, и задаются с помощью первого аргумента функции. (Используйте help(axis) для получения полного списка параметров.)

> x <- rnorm(10,mean=0,sd=4)
> y <- 3*x-1+rnorm(10,mean=0,sd=2)
   summary(x)
    Min. 1st Qu.  Median    Mean 3rd Qu.    Max.
 -6.1550 -1.9280  1.2000 -0.1425  2.4780  3.1630
> summary(y)
    Min.  1st Qu.   Median     Mean  3rd Qu.     Max.
-17.9800  -9.0060   0.7057  -1.2060   8.2600  10.9200
> plot(x,y,axes=FALSE,col=2)
> axis(1,pos=c(0,0),at=seq(-7,5,by=1))
> axis(2,pos=c(0,0),at=seq(-18,11,by=2))
> 

Команда par  может быть использована для установки значения по умолчанию для различных параметров. В приведенном ниже примере, фон по умолчанию установлен в серый, отсутствие границы вокруг окна, и поля для осей будет в два раза меньше обычного.

> par(bty="l")
> par(bg="gray")
> par(mex=2)
> x <- rnorm(10,mean=0,sd=4)
> y <- 3*x-1+rnorm(10,mean=0,sd=2)
> plot(x,y)
> 

Еще одна распространенная задача: разместить текстовую строку на графике. Функция text() получает координаты и строку, и ставит строку на заданные координаты. Функция text() имеет опции для настройки смещения, размера, шрифта и другие параметры. В приведенном ниже примере слой "numbers!" размещается на графике. Используйте help(text), чтобы увидеть больше параметров.

> x <- rnorm(10,mean=0,sd=4)
> y <- 3*x-1+rnorm(10,mean=0,sd=2)
> plot(x,y)
> text(-1,-2,"numbers!")
> 

По умолчанию, функция text обрезает любые символы за пределами области графика. Этот режим может быть изменен с помощью аргумента xpd.

> x <- rnorm(10,mean=0,sd=4)
> y <- 3*x-1+rnorm(10,mean=0,sd=2)
> plot(x,y)
> text(-7,-2,"outside the area",xpd=TRUE)
> 

3. Индексация векторов

3.1. Логическая индексация 

Сначала приведем пример того, как выбрать отдельные элементы в вектор. Первым этапом является определение вектора данных, второй этап состоит в определении вектора из логических значений (TRUE, FALSE). Когда вектор логических значений используется для индекса в векторе значений, то возвращаются только элементы, соответствующие переменным, которые индексированы по TRUE:

> a <- c(1,2,3,4,5)
> b <- c(TRUE,FALSE,FALSE,TRUE,FALSE)
> a[b]
[1] 1 4
> max(a[b])
[1] 4
> sum(a[b])
[1] 5

3.2. Отсутствующие или недостающие значения 

Существует предопределенная переменная, обозначаемая NA, она может быть использована для того, чтобы указать отсутствующую информацию. Проблема в том, что некоторые функции выдадут ошибку, если одна из записей в данных отмечена NA. Некоторые функции позволяют игнорировать недостающие значения используя специальные :

 

> a <- c(1,2,3,4,NA)
> a
[1]  1  2  3  4 NA
> sum(a)
[1] NA
> sum(a,na.rm=TRUE)
[1] 10

Есть другие варианты, например когда эта функция не доступна, или вы просто хотите пропустить эти значения. Функция is.na может быть использована, чтобы определить, какие значения не доступны. Логический оператор "нет" в R является символ “!”. При использовании индексации значения внутри вектора, которые помечены NA могут быть также легко удалены:

> a <- c(1,2,3,4,NA)
> is.na(a)
[1] FALSE FALSE FALSE FALSE  TRUE
> !is.na(a)
[1]  TRUE  TRUE  TRUE  TRUE FALSE
> a[!is.na(a)]
[1] 1 2 3 4
> b <- a[!is.na(a)]
> b
[1] 1 2 3 4

3.3. Индексирование логическими выражениями 

Любое логическое выражение может быть использовано в качестве индекса, который открывает широкие возможности. Например, вы можете удалить или сосредоточиться на записи, которые соответствуют определенным критериям. Например, вы, возможно, захотите удалить все записи, которые выше определенного значения:

> a = c(6,2,5,3,8,2)
> a
[1] 6 2 5 3 8 2
> b = a[a<6]
> b
[1] 2 5 3 2

В качестве другого примера, предположим, что вы хотите, объединить значения, которые соответствуют двум различным факторам в другой вектор:

> d = data.frame(one=as.factor(c('a','a','b','b','c','c')),
two=c(1,2,3,4,5,6))
> d
  one two
1   a   1
2   a   2
3   b   3
4   b   4
5   c   5
6   c   6
> both = d$two[(d$one=='a') | (d$one=='b')]
> both
[1] 1 2 3 4

Следует отметить оператор  '|', который был использован в предыдущем примере. Существует разница между "||" и "|".  Одиночный бар будет выполнять векторную операцию почленно, а двойной бар будет выражаться как TRUE или FALSE результат:

> (c(TRUE,TRUE))|(c(FALSE,TRUE))
[1] TRUE TRUE
> (c(TRUE,TRUE))||(c(FALSE,TRUE))
[1] TRUE
> (c(TRUE,TRUE))&(c(FALSE,TRUE))
[1] FALSE  TRUE
> (c(TRUE,TRUE))&&(c(FALSE,TRUE))
[1] FALSE
